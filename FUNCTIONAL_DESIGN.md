# MiMo Studio — 功能说明 & UX 优化方案

## 一、当前功能模块说明

MiMo Studio 是一个角色配音工具，核心流程是：**选择声音 → 编写剧本 → 控制表达 → 生成音频**。

### 模块 1：角色管理（左侧栏）

**定位**：「声音预设库」—— 把一套声音配置（音色 + 模型 + 风格）打包保存，方便复用。

- 每个角色 = 一组声音配置的快照
- 点击角色 → 自动填充右侧「声音设置」面板
- 角色可选，也可不选（直接用右侧面板即兴配置）

### 模块 2：剧本编辑（中间区域）

**定位**：「台词输入区」—— 你在这里写需要配音的文本。

- 每行一条台词
- 支持**行内标签**即时切换表达方式，例如：
  - `(开心)今天是个好日子` → 这句用开心的语气
  - `(东北话)哎呀妈呀` → 这句切换方言
  - `[emotion: excited]` → 英文情绪标签

### 模块 3：标签栏（中间下方）

**定位**：「表达控制工具箱」—— 点击即可插入行内标签。

- **情感**：(开心) (悲伤) (愤怒) (激动) …
- **方言**：(台湾腔) (东北话) (粤语) …
- **语速**：(变快) (变慢) …
- **语气**：(耳语) (喊叫) (轻声) …
- **声线**：(少女声线) (成熟女声) …

### 模块 4：声音设置（右侧栏）

**定位**：「声音 & 风格配置」—— 决定「谁在说话」和「怎么说话」。

| 配置项 | 作用 | 类比 |
|--------|------|------|
| 模型 | 选择 TTS 引擎（预置音色 / 自定义音色 / 克隆音色） | 选配音演员类型 |
| 预置音色 | 8 个内置声音（冰糖、Mia 等） | 选具体配音演员 |
| 声音描述 | Voice Design 模型专用：用自然语言描述想要的音色 | "我要温柔知性的御姐音" |
| 风格提示词 | 全局播报风格（情绪基调、语速） | 导演给演员的表演指导 |

### 模块 3 和 4 的区别（重要）

| | 标签栏（模块 3） | 风格提示词（模块 4） |
|---|---|---|
| 作用范围 | **单句/短语**级别 | **全局**（整段文本） |
| 用法 | 插入在文本中，即时切换 | 一次性设定，影响全文基调 |
| 例子 | `(开心)太好了！(悲伤)但是…` | "用轻快上扬的语调，声音明亮" |

## 二、当前 UX 问题诊断

### 问题 A：三栏布局缺乏流程引导
用户看到三栏并列，不清楚从哪里开始——是先创建角色？先选声音？还是直接写剧本？缺少**视觉流线**引导操作顺序。

### 问题 B：角色和声音设置的关系模糊
角色 = 声音设置的快捷方式，但这个关系不够直观。选中角色后，右侧面板的修改是「覆盖角色配置」还是「独立于角色」？用户困惑。

### 问题 C：标签栏和风格提示词的功能重叠感
两者都控制「怎么说话」，但一个是行内、一个是全局。当前 UI 把它们放在不同位置（中间下方 vs 右侧），用户需要上下左右来回看。

### 问题 D：生成按钮孤立
Generate 按钮在右侧栏底部，与剧本编辑区（中间）和音频播放区（中间下方）距离较远，操作流被割裂。

## 三、优化方案

### 方案概述：从「三栏仪表盘」重构为「线性工作流」

**核心理念**：用户配音的思维过程是线性的——
1. 选谁说话 → 2. 写什么内容 → 3. 怎么表达 → 4. 生成 & 听结果

把 UI 从上到下组织，而不是左中右分栏。

### 新布局结构

```
┌─────────────────────────────────────────────┐
│  HEADER（标题、语言、主题切换、API状态）      │
├─────────────────────────────────────────────┤
│                                             │
│  STEP 1 — 声音配置（可折叠）                  │
│  ┌──────────────────────────────────────┐   │
│  │ [角色快捷选择: 冰糖·女主] [或手动配置▾] │   │
│  │ 模型 · 音色 · 风格提示词               │   │
│  └──────────────────────────────────────┘   │
│                                             │
│  STEP 2 — 剧本编辑                          │
│  ┌──────────────────────────────────────┐   │
│  │ [标签快捷栏: 情感▾ 方言▾ 语速▾ ...]    │   │
│  │                                      │   │
│  │ ┌────────────────────────────────┐   │   │
│  │ │ (开心)今天的天气真不错啊。       │   │   │
│  │ │ (东北话)可不是嘛，老好了！       │   │   │
│  │ │                                │   │   │
│  │ └────────────────────────────────┘   │   │
│  │                                      │   │
│  │ [🔊 生成音频]  ←── 紧贴文本区下方      │   │
│  └──────────────────────────────────────┘   │
│                                             │
│  STEP 3 — 播放 & 下载（生成后出现）           │
│  ┌──────────────────────────────────────┐   │
│  │ ▸▸▸▸▸▸▸ 波形可视化  ▸▸▸▸▸▸▸  ⬇下载 │   │
│  └──────────────────────────────────────┘   │
│                                             │
│  SIDEBAR — 角色库（左侧可折叠）                │
│  ┌────────────────────┐                     │
│  │ + 新建角色          │                     │
│  │ ────────────────── │                     │
│  │ ● 傲娇女主角        │                     │
│  │   冰糖 · 开心快语速  │                     │
│  │ ● 沉稳旁白音        │                     │
│  │   白桦 · 严肃纪录片  │                     │
│  └────────────────────┘                     │
└─────────────────────────────────────────────┘
```

### 关键改动

#### 1. 角色快捷栏（替代分散的角色管理）
- 顶部一个**横向角色条**：显示已选角色名称 + 快速切换下拉
- 未选角色时显示「选择一个角色预设，或手动配置 ▾」
- 选中的角色高亮，展开可看详情、可编辑
- 右侧的「保存当前配置为角色」按钮

#### 2. 声音设置内嵌在角色栏下方（默认折叠）
- 点击「手动配置 ▾」展开
- 包含：模型选择、音色选择、风格提示词、声音描述
- 当角色选中时，展开显示「当前使用 [角色名] 的配置，修改将临时覆盖」

#### 3. 标签栏移到文本区上方
- 标签栏紧贴文本输入框上方，视觉上形成「先选标签 → 再写文本」的引导
- 标签按类别横向排列为小型 chips，点击插入到光标位置

#### 4. 生成按钮移到文本区正下方
- 「生成音频」按钮紧贴文本区下方，形成「写 → 点 → 听」的直觉流
- 加载状态在按钮内显示

#### 5. 角色管理改为可折叠左侧抽屉
- 默认折叠（小图标），hover 或点击展开
- 不占用主工作区空间
- 展开后可以管理（增删改）角色
